# Milestone 2.8: Next Round Fixes

From goat-audit of all uncommitted changes (71 files, 3379 insertions). 11 findings after 4-pass audit.

## High

### 1. Skill frontmatter inconsistency across .claude/ and .agents/

**Files:** All 7 skill pairs — `.claude/skills/goat-*/SKILL.md` vs `.agents/skills/goat-*/SKILL.md`
**Issue:** `.agents/` has `name:` field and `# GOAT Name` title. `.claude/` has no `name:` field and `# /goat-name` title. All 7 pairs differ.
**Root cause:** `.agents/skills/` was created first with `name:` frontmatter for Codex discovery. `.claude/skills/` was created separately with Claude's `/skill-name` title convention. Never unified.
**Fix:** Decide on one format. Recommendation: add `name:` to `.claude/` files (CI already validates it). Keep both title formats since they match agent conventions.
- [ ] Add `name: goat-{skill}` to all 7 `.claude/skills/` frontmatter blocks
- [ ] Verify CI context-validate.sh passes after change

### 2. Non-null assertions on optional Detection fields

**File:** `src/cli/evaluate/evaluators.ts:165-166,287`
**Issue:** `detect.pass!` and `detect.fail!` use non-null assertions on optional fields. Currently safe (only `line_count` and `count_items` checks define these) but will crash if a new check type omits them.
**Fix:** Add defaults:
- [ ] Change `detect.pass!` to `detect.pass ?? 0`
- [ ] Change `detect.fail!` to `detect.fail ?? Infinity`
- [ ] Same for `detect.min!` in `evalGrepCount`
