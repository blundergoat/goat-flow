## Phase B: ESLint Warning Cleanup

### Non-null assertions (15 warnings)

> **Note:** Line numbers below are approximate — JSDoc comments were added after these were recorded. Use the function/variable name to locate, not the line number.

Replace `!` assertions with proper type narrowing. These are in files where regex capture groups or object lookups are checked by an `if` but TypeScript doesn't narrow the indexed result:

- [ ] `src/cli/eval/parser.ts` — 4 instances (lines 136, 145, 163, 228): regex `match[1]!` after null check → extract to typed variable
- [ ] `src/cli/facts/agent.ts` — 6 instances: `headingMatch[1]!`, `match[1]!` in matchAll loops → guard or destructure
- [ ] `src/cli/facts/shared.ts` — 1 instance (line 18): `match[1]!` → guard
- [ ] `src/cli/facts/fs.ts:84` — 1 instance: `parts[patternIndex]!` → already bounds-checked, use guard
- [ ] `src/cli/evaluate/evaluators.ts:346` — 1 instance: `pathMap[path]!` after `in` check → use optional chaining with fallback
- [ ] `src/cli/prompt/compose-fix.ts:63` — 1 instance: `agentOverrides[agentId]!` → optional chain
- [ ] `src/cli/prompt/compose-setup.ts:32` — 1 instance: `agentOverrides[agentId]!` → optional chain

**Approach:** For regex matches, extract a helper:
```typescript
function captureGroup(match: RegExpMatchArray, index: number): string {
  const value = match[index];
  if (value === undefined) throw new Error(`Capture group ${index} missing`);
  return value;
}
```

For agent override lookups, replace:
```typescript
// Before
if (f.agentOverrides?.[agentId]) {
  instruction = f.agentOverrides[agentId]!;
}
// After
const override = f.agentOverrides?.[agentId];
if (override) instruction = override;
```
